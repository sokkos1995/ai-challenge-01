# Сравнение ответов: без ограничений vs с ограничениями

Ниже показан пример одного и того же запроса в двух режимах.
Смысл сравнения: увидеть, как ограничения делают ответ более предсказуемым и управляемым.

## Запрос

`Объясни, что такое LLM`

## 1) Без ограничений

Команда:

```bash
python3 llm_cli.py -v "Объясни, что такое LLM"
```

Типичный ответ:

> LLM (Large Language Model) — это большая языковая модель, обученная на огромных объемах текста.  
> Она умеет отвечать на вопросы, писать тексты, переводить и суммировать, но иногда может ошибаться и выдавать неточные факты.

Что видно:
- формат свободный;
- длина может плавать от короткого абзаца до длинного объяснения;
- финал ответа нефиксированный.

## 2) С ограничениями (формат + длина + завершение)

Команда:

```bash
python3 llm_cli.py -v \
  --response-format 'JSON: {"definition": string, "risks": [string]}' \
  --max-output-tokens 10000 \
  --stop-sequence "<END>" \
  --finish-instruction "Заверши ответ строкой <END>" \
  "Объясни, что такое LLM"
```

Типичный ответ:

```json
{"definition":"LLM — большая языковая модель, предсказывающая следующий токен и генерирующая текст.","risks":["может галлюцинировать","может наследовать смещения данных"]}
```

`<END>`

Что видно:
- формат задан явно и соблюдается;
- ответ короче за счет лимита;
- есть контролируемое завершение (`<END>`).

## Вывод

Ограничения полезны, когда нужен:
- стабильный формат для парсинга;
- предсказуемая длина ответа;
- явный признак окончания генерации.
