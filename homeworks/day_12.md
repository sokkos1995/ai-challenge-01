🔥 День 12. Персонализация ассистента

Реализована персонализация поверх memory layers из `day_11`.

## Что добавлено

- отдельное SQLite-хранилище пользователей в скрытой папке `.llm_users/`;
- флаг `--user-id` для запуска диалога в контексте конкретного пользователя;
- автоматическое создание нового пользователя при первом входе;
- входное мини-интервью для новых пользователей:
  - `role`
  - `stack`
  - `answer_detail` (`brief` / `detailed`)
  - `answer_format`
  - `constraints`
- команда `@personalization` для просмотра и пополнения профиля;
- автоматическая подстановка профиля пользователя в `system prompt` каждого запроса.

## Как это работает

### 1. Запуск под пользователем

```bash
python3 llm_cli.py --chat --context-strategy memory --user-id 123
```

Если `user_id=123` не существует, агент:
- создает запись пользователя;
- задает мини-интервью;
- сохраняет ответы в `.llm_users/users.db`.

### 2. Раздельное хранение по пользователям

Для каждого `user_id` создается отдельная сессия:

- своя история чата;
- свои memory layers;
- свой профиль персонализации.

Это позволяет запускать разные диалоги с разными настройками ответа.

### 3. Дополнение персонализации

Внутри чата доступны команды:

```text
@personalization show
@personalization interview
@personalization constraints=Не предлагай JavaScript
@personalization answer_format=step-by-step
```

## Что проверить

### Короткий демо-сценарий

1. Запустите чат для пользователя `123` и заполните профиль backend-разработчика с краткими ответами.
2. Задайте вопрос: `Объясни, как сделать JWT auth.`
3. Запустите отдельный чат для пользователя `456` и заполните профиль junior analyst с подробными пошаговыми ответами.
4. Задайте тот же вопрос: `Объясни, как сделать JWT auth.`
5. Сравните ответы.

Проверьте:
- для разных профилей ответы отличаются по стилю, детализации и формату;
- ассистент автоматически учитывает `role`, `stack`, `answer_detail`, `answer_format`, `constraints`;
- во втором сообщении не нужно заново писать: `отвечай кратко` или `объясняй пошагово` - профиль уже подставляется сам.

### Профиль A

```bash
python3 llm_cli.py --chat --context-strategy memory --user-id 123
```

Пример интервью:
- `role=backend developer`
- `stack=Python, FastAPI, Postgres`
- `answer_detail=brief`
- `answer_format=bullets`
- `constraints=без воды`

Спросите:

```text
Объясни, как сделать JWT auth.
```

Ожидание: ответ короткий, списком, с учетом backend/Python контекста.

### Профиль B

```bash
python3 llm_cli.py --chat --context-strategy memory --user-id 456
```

Пример интервью:
- `role=junior analyst`
- `stack=SQL, Excel`
- `answer_detail=detailed`
- `answer_format=step-by-step`
- `constraints=объясняй простыми словами`

Задайте тот же вопрос.

Ожидание: ответ будет подробнее, проще и в пошаговом формате.

### Что должно быть видно автоматически

После интервью в каждом профиле задайте еще один короткий запрос без дополнительных инструкций:

```text
Дай план решения.
```

Ожидание:
- для `user_id=123` ответ останется коротким и в виде списка;
- для `user_id=456` ответ будет более подробным и пошаговым;
- ассистент сам сохранит выбранный стиль, даже если в текущем сообщении это не повторялось.

## Результат

Получился персонализированный агент, который:
- хранит профиль пользователя отдельно;
- автоматически учитывает предпочтения пользователя в новых запросах;
- совмещает персонализацию с memory layers и историей диалога.
