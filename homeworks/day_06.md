# Простой LLM-агент с инкапсуляцией API (CLI-демо)

🔥 День 6. Простой агент

Реализуйте простого агента, который:  
👉 принимает запрос пользователя  
👉 отправляет его в LLM через API  
👉 получает ответ  
👉 выводит результат в вашем интерфейсе

(простой чат, CLI или web, запросы через HTTP-клиент)

Важно:  
👉 агент должен быть отдельной сущностью, а не просто один вызов API  
👉 логика запроса и ответа должна быть инкапсулирована в агенте

Результат:  
Агент принимает запрос и корректно вызывает LLM через API

---

## Что реализовано

- **Интерфейс:** CLI (single prompt + интерактивный chat-режим).
- **Отдельная сущность агента:** класс `SimpleLLMAgent` в `app/agent.py`.
- **Инкапсуляция логики:** внутри агента скрыты:
  - выбор провайдера/модели из окружения;
  - сборка HTTP-запроса (`urllib.request`);
  - отправка в Chat Completions API;
  - обработка ответа и ошибок;
  - fallback на запасные модели.
- **Точка входа:** `llm_cli.py` → `app/cli.py`, где пользовательский ввод передаётся в `agent.ask(...)`.

---

## Где в коде видно требования задания

### 1) Агент как отдельная сущность

- `app/agent.py`: класс `SimpleLLMAgent`.
- Создание агента: `SimpleLLMAgent.from_env()`.
- Запрос к агенту: `agent.ask(prompt, options)`.

### 2) Логика запроса/ответа инкапсулирована в агенте

- Подготовка payload и `messages` делается внутри `_send_request(...)` (`app/agent.py`).
- HTTP POST в LLM API (`urllib.request.Request`, `urlopen`) тоже внутри `app/agent.py`.
- Парсинг `choices[0].message.content`, измерение latency и обработка ошибок — внутри `SimpleLLMAgent.ask(...)`.

### 3) Агент принимает запрос и выводит ответ в интерфейсе

- `app/cli.py`:
  - в обычном режиме берёт prompt и печатает `response.answer`;
  - в `--chat` режиме читает `you> ...`, вызывает `agent.ask(...)`, печатает `agent> ...`.

## Как запустить демо

Из корня проекта `hw01/`.

1) Подготовить `.env` с ключом (пример для OpenRouter):

```bash
cat > .env <<'EOF'
OPENROUTER_API_KEY=ваш_api_ключ
EOF
```

2) Одиночный запрос:

```bash
python3 llm_cli.py "Объясни, что такое агент в LLM-приложении в 2 предложениях"
```

3) Интерактивный чат:

```bash
python3 llm_cli.py --chat
```

Пример диалога:

```text
you> Привет! Что ты умеешь?
agent> ...
you> exit
Bye!
```

## Проверка результата по критериям задания

- ✅ **Принимает запрос пользователя:** через аргумент CLI или в интерактивном режиме.
- ✅ **Отправляет запрос в LLM через API:** HTTP POST в Chat Completions endpoint.
- ✅ **Получает ответ:** извлекает `message.content` из `choices`.
- ✅ **Выводит результат в интерфейсе:** печатает ответ в консоль.
- ✅ **Агент — отдельная сущность:** `SimpleLLMAgent`.
- ✅ **Логика инкапсулирована:** API-взаимодействие находится в `app/agent.py`, а не размазано по интерфейсу.

Итог: требование «Агент принимает запрос и корректно вызывает LLM через API» выполнено.
