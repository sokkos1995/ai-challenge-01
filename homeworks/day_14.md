🔥 День 14. Инварианты и ограничения состояния

Добавьте в ассистента инварианты, которые он не имеет права нарушать.

Примеры инвариантов:

👉 выбранная архитектура
👉 принятые технические решения
👉 ограничения по стеку
👉 бизнес-правила

Сделайте так, чтобы:

👉 инварианты хранились отдельно от диалога
👉 ассистент явно учитывал их в рассуждениях
👉 ассистент отказывался предлагать решения, которые их нарушают

Проверьте:

👉 что происходит при конфликте запроса и инварианта
👉 как ассистент объясняет отказ

Результат:

Ассистент, который работает в рамках заданных инвариантов

Реализация:

- инварианты вынесены в отдельный persistent layer `long_term.invariants` и хранятся отдельно от диалога в SQLite;
- инварианты подмешиваются в system prompt как отдельный источник истины;
- перед обычным ответом запускается `invariant guard`: он проверяет запрос на конфликт с инвариантами;
- если запрос конфликтует с инвариантом, ассистент не предлагает нарушающее решение, а возвращает отказ с объяснением и безопасной альтернативой.

Минимальный план доработки:

1. Добавить отдельное хранилище `invariants` в long-term memory.
2. Подмешивать инварианты в prompt как non-negotiable constraints.
3. Вставить guard-проверку перед генерацией ответа.
4. Добавить CLI-команды для демо: `@invariant show|add|clear`.

Демо:

```text
$ python3 llm_cli.py --chat --context-strategy memory

you> @invariant clear
agent> invariants cleared.

you> @invariant add Архитектура: backend остается на Python, без миграции на Node.js.
agent> invariant saved.

you> @invariant add Бизнес-правило: персональные данные нельзя выносить во внешние SaaS.
agent> invariant saved.
you> @invariant show
agent> invariants:
[
  "Архитектура: backend остается на Python, без миграции на Node.js.",
  "Бизнес-правило: персональные данные нельзя выносить во внешние SaaS."
]

you> Предложи план миграции backend на Node.js и отправки персональных данных во внешний SaaS.
agent> Не могу предложить это решение: оно нарушает сохраненные инварианты.
Конфликтующий инвариант:
- Архитектура: backend остается на Python, без миграции на Node.js.
- Бизнес-правило: персональные данные нельзя выносить во внешние SaaS.
Запрос нарушает два инварианта: миграцию backend на Node.js и вынос персональных данных во внешний SaaS.
Могу предложить безопасную альтернативу: Предложите план по развитию текущего backend на Python или по интеграции с внешними сервисами, не нарушая при этом правила безопасности данных.
```

Проверка, что инварианты живут отдельно от диалога и сохраняются после перезапуска:

```text
$ python3 llm_cli.py --chat --context-strategy memory

you> @invariant show
agent> invariants:
[
  "Архитектура: backend остается на Python, без миграции на Node.js.",
  "Бизнес-правило: персональные данные нельзя выносить во внешние SaaS."
]
```

Что происходит при конфликте запроса и инварианта:

- запрос сначала проходит через отдельную guard-проверку;
- guard сравнивает запрос со списком сохраненных инвариантов;
- если выполнение запроса требует нарушить инвариант, обычная генерация ответа не запускается;
- вместо этого ассистент сразу возвращает отказ.

Как ассистент объясняет отказ:

- явно пишет, что не может предложить решение из-за нарушения инвариантов;
- перечисляет, какой именно инвариант конфликтует с запросом;
- коротко объясняет причину конфликта;
- предлагает безопасную альтернативу, которая остается в рамках заданных ограничений.
