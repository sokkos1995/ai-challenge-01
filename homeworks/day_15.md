# 🔥 День 15. Контролируемые переходы состояний

Реализуйте явные переходы между состояниями задачи.

Сделайте так, чтобы:

👉 у задачи были допустимые состояния
👉 были разрешённые переходы между ними
👉 ассистент не мог “перепрыгнуть” этап

Пример:

👉 нельзя делать реализацию до утверждённого плана
👉 нельзя делать финал без валидации

Проверьте:

👉 попытки перейти в недопустимое состояние
👉 реакцию ассистента
👉 корректность продолжения после паузы

Результат:

Ассистент с контролируемым жизненным циклом задачи

## Реализация:

- в `working memory` добавлены явные поля контроля этапов: `state`, `plan_status`, `validation_status`, `paused`;
- переходы между стадиями проверяются через `task_state_machine`, а не только через свободное изменение поля;
- переход `PLANNING -> EXECUTION` разрешен только если план не пустой и явно утвержден;
- переход `VALIDATION -> DONE` разрешен только если валидация завершилась со статусом `PASSED`;
- в `memory`-режиме отдельный lifecycle guard останавливает ассистента, если пользователь просит перепрыгнуть этап;
- состояние задачи сохраняется в SQLite, поэтому после паузы и перезапуска можно продолжить с того же места.

Демо:

```text
$ python3 llm_cli.py --chat --context-strategy memory

you> @mem clear work
agent> memory layer cleared: work

you> @mem work task=Реализовать контролируемый жизненный цикл задачи
agent> working memory updated: task

you> @task plan+ Описать допустимые состояния и переходы
agent> task plan updated.

you> @task plan+ Добавить guard для запрета перепрыгивания этапов
agent> task plan updated.

you> @task show
agent> task:
{
  "task": "Реализовать контролируемый жизненный цикл задачи",
  "state": "PLANNING",
  "plan_status": "DRAFT",
  "validation_status": "PENDING",
  "paused": false,
  "step": 0,
  "total": 0,
  "expected_action": "",
  "allowed_transitions": ["EXECUTION"],
  "plan": [
    "Описать допустимые состояния и переходы",
    "Добавить guard для запрета перепрыгивания этапов"
  ],
  "done": [],
  "notes": []
}

Проверьте:
👉 попытки перейти в недопустимое состояние
👉 реакцию ассистента
👉 корректность продолжения после паузы

1) Попытка перейти в недопустимое состояние

you> @task state EXECUTION
agent> task command error: Cannot start execution: the plan is not approved yet.

you> Сделай реализацию прямо сейчас
agent> Не могу выполнить этот шаг: он нарушает жизненный цикл задачи.
Нельзя переходить к реализации на стадии PLANNING. Сначала выполните допустимый переход в EXECUTION.
Следующий корректный шаг: Сначала подготовьте и утвердите план, затем переведите задачу в EXECUTION.

2) Корректный переход после утверждения плана

you> @task approve-plan
agent> task plan approved.

you> @task state EXECUTION
agent> task state updated: EXECUTION

you> @task done+ Реализован task state machine с явными переходами
agent> task done updated.

you> @task state VALIDATION
agent> task state updated: VALIDATION

3) Попытка сделать финал без успешной валидации

you> @task state DONE
agent> task command error: Cannot finalize the task: validation has not passed yet.

you> Сделай финальный ответ
agent> Не могу выполнить этот шаг: он нарушает жизненный цикл задачи.
Нельзя делать финальный ответ до завершения обязательной валидации и перехода в DONE.
Следующий корректный шаг: Сначала завершите валидацию и только после этого переходите к финалу.

4) Пауза и корректное продолжение

you> @task pause
agent> task paused.

you> Давай сразу продолжим реализацию
agent> Не могу выполнить этот шаг: он нарушает жизненный цикл задачи.
Задача сейчас на паузе, поэтому нельзя продолжать следующий этап.
Следующий корректный шаг: Сначала возобновите задачу, затем выполните следующий допустимый переход.

you> @task resume
agent> task resumed.

you> @task validate pass
agent> task validation marked as passed.

you> @task state DONE
agent> task state updated: DONE

you> Сделай финальный ответ
agent> Краткий итог: реализован контролируемый жизненный цикл задачи с допустимыми состояниями,
разрешенными переходами, проверкой утверждения плана, обязательной валидацией перед финалом
и корректным восстановлением после паузы.
```
