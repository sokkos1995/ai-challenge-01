Результат:

## Что реализовано

- **Хранилище:** после каждого ответа в режиме `--chat` история пишется в **SQLite** (один файл БД на диске).
- **Схема:** таблица **`chat_messages`** (`id`, `role` — только `user` / `assistant`, `content`). При сохранении выполняется полная перезапись содержимого таблицы актуальным списком реплик (транзакция `BEGIN IMMEDIATE`).
- **Путь к файлу БД:** переменная окружения **`LLM_CHAT_HISTORY_PATH`** (по умолчанию **`.llm_chat_history.db`** в текущей рабочей директории — обычно корень `hw01/`, откуда запускается `llm_cli.py`).
- **Загрузка:** при первом вызове `ask_chat` сообщения читаются из БД **в порядке `id`** (если файл уже существует).
- **Отключение персистентности:** задайте пустое значение, например `LLM_CHAT_HISTORY_PATH=` в `.env`, тогда путь станет «пустым» и история не пишется на диск (только в памяти процесса).

Функции **`_init_chat_db`**, **`load_chat_messages`** / **`save_chat_messages`** (через `sqlite3`), метод **`ask_chat`** и поле **`chat_history_path`** — в `app/agent.py`. При старте чата CLI печатает абсолютный путь к файлу SQLite, если персистентность включена: `Chat history SQLite (restored on restart): …` (`app/cli.py`).

---

## Где в коде видно требования

| Требование | Место в коде |
|------------|----------------|
| Схема и создание таблицы | `_init_chat_db`, `CREATE TABLE IF NOT EXISTS chat_messages` в `app/agent.py` |
| Сохранение после реплик | `SimpleLLMAgent.ask_chat` → `save_chat_messages` (очистка + `INSERT`, `COMMIT`) |
| Восстановление при новом запуске | `_ensure_chat_history_loaded` → `load_chat_messages` (`SELECT … ORDER BY id`) |
| Путь из окружения | `_chat_history_path_from_env()` (дефолт `.llm_chat_history.db`), `SimpleLLMAgent.from_env()` |

## Как пройти демо (пошагово)

Из корня проекта **`hw01/`** (там же `llm_cli.py` и `.env` с ключом API).

1) **Первый запуск — начать диалог**

```bash
python3 llm_cli.py --chat
```

В начале будет строка вида `Chat history SQLite (restored on restart): /…/hw01/.llm_chat_history.db` — это подтверждает путь к файлу БД.

Введите что-то **однозначное**, что модель должна «помнить» в следующей сессии, например:

```text
you> Запомни кодовое слово на сегодня: синий слон. Ничего больше не добавляй, просто подтверди.
agent> …
you> exit
Bye!
```

2) **Перезапуск приложения**

Снова из того же каталога:

```bash
python3 llm_cli.py --chat
```

История подтянется из того же файла SQLite (путь тот же).

3) **Продолжить диалог и проверить память**

```text
you> Какое кодовое слово я просил запомнить в прошлый раз?
agent> …
```

Ожидаемо: ответ ссылается на **«синий слон»** (или эквивалент), потому что в запрос к API уходят восстановленные сообщения `user` / `assistant` из БД.

При желании посмотрите содержимое БД (из корня `hw01/`):

```bash
sqlite3 .llm_chat_history.db "SELECT id, role, substr(content, 1, 80) FROM chat_messages ORDER BY id;"
```
