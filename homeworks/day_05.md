# Сравнение слабой, средней и сильной моделей (Hugging Face → OpenRouter)

🔥 День 5. Версии моделей

Выполните один и тот же запрос:
👉 на слабой модели
👉 на средней модели
👉 на сильной модели

(например: из начала, середины и конца списка HuggingFace)

Замерьте:
👉 время ответа
👉 количество токенов
👉 стоимость (если модель платная)

Короткий вывод о различиях между моделями + ссылки

## Условия эксперимента

- **Один и тот же промпт** (задача на условную вероятность; формулировка в [`task.txt`](../task.txt), развёрнутый вариант с инструкцией — в [`prompt.txt`](../prompt.txt)): три коробки с монетами, случайный выбор коробки и монеты; при условии, что монета золотая, найти \(P(A \mid \text{золотая})\).
- **Провайдер:** [OpenRouter](https://openrouter.ai/) (единый Chat Completions API).
- **Инструмент:** `python3 llm_cli.py -v --temperature 0.2 …` — время считается до получения полного ответа; токены и стоимость берутся из поля `usage` ответа API. Промпт удобно передавать из файла флагом **`-f`** (см. раздел «Примеры команд»). Готовые команды для трёх моделей — там же.

### Выбор «начало / середина / конец» списка HF

На [странице моделей Hugging Face](https://huggingface.co/models) при сортировке по популярности в топе оказываются в основном крупные семейства (Llama, Qwen, Mistral и т.д.). ## Примеры команд

Использовался скрипт [`llm_cli.py`](../llm_cli.py) из корня репозитория. Ключ OpenRouter задаётся в `.env` (`OPENROUTER_API_KEY` или `LLM_API_KEY` при провайдере OpenRouter); скрипт подхватывает `.env` из **текущей рабочей директории** при запуске.

Для замеров по конкретной модели без автопереключения на фолбэки: **`LLM_FALLBACK_MODELS=""`**.

**Промпт из файла (`-f`):** один и тот же текст для всех трёх моделей — из [`prompt.txt`](../prompt.txt) в корне репозитория (альтернатива — переменная **`LLM_PROMPT_FILE=prompt.txt`** в `.env`, тогда флаг `-f` можно не писать).

**Запуск из корня проекта** (`hw01/`, там же `llm_cli.py`, `.env` и `prompt.txt`):

```bash
# из каталога, где лежат llm_cli.py и .env (корень репозитория hw01)

LLM_MODEL="meta-llama/llama-3.2-1b-instruct" LLM_FALLBACK_MODELS="" \
  python3 llm_cli.py -v --temperature 0.2 -f prompt.txt

LLM_MODEL="qwen/qwen-2.5-7b-instruct" LLM_FALLBACK_MODELS="" \
  python3 llm_cli.py -v --temperature 0.2 -f prompt.txt

LLM_MODEL="qwen/qwen-2.5-72b-instruct" LLM_FALLBACK_MODELS="" \
  python3 llm_cli.py -v --temperature 0.2 -f prompt.txt
```

**Последовательный цикл** (все три замера подряд):

```bash
# из каталога, где лежат llm_cli.py и .env

for MODEL in "meta-llama/llama-3.2-1b-instruct" "qwen/qwen-2.5-7b-instruct" "qwen/qwen-2.5-72b-instruct"; do
  echo "========== MODEL: $MODEL =========="
  LLM_MODEL="$MODEL" LLM_FALLBACK_MODELS="" python3 llm_cli.py -v --temperature 0.2 -f prompt.txt 2>&1
  echo ""
done
```

- Ответ модели печатается в **stdout**, блок `[stats] …` (латентность, токены, `cost`, `finish_reason`) — в **stderr**; `2>&1` в цикле объединяет потоки в одном логе.
- Если запускать из каталога `homeworks/`, укажите пути к скрипту и к файлу промпта:

```bash
cd homeworks
LLM_MODEL="qwen/qwen-2.5-7b-instruct" LLM_FALLBACK_MODELS="" \
  python3 ../llm_cli.py -v --temperature 0.2 -f ../prompt.txt
```

**Альтернатива без файла** — одна строка в переменной (как в ранних замерах; при использовании `-f` она не нужна):

```bash
PROMPT='Есть 3 коробки: A — 2 золотые монеты, B — 1 золотая и 1 серебряная, C — 2 серебряные. Коробку выбирают наугад (1/3), затем наугад достают монету. Монета оказалась золотой. Найди P(A|золотая). Ответ: дробь и проценты, кратко покажи ход.'

LLM_MODEL="qwen/qwen-2.5-7b-instruct" LLM_FALLBACK_MODELS="" \
  python3 llm_cli.py -v --temperature 0.2 "$PROMPT"
```

## Замеры

| Модель | Латентность (мс) | prompt / completion / всего токенов | Стоимость (USD, из ответа API) |
|--------|------------------|-------------------------------------|--------------------------------|
| Llama 3.2 **1B** | 1 925 | 109 / 277 / **386** | ≈ **0.000058** (`5.83e-05`) |
| Qwen 2.5 **7B** | 30 154 | 140 / 739 / **879** | ≈ **0.000080** (`7.95e-05`) |
| Qwen 2.5 **72B** | 17 011 | 119 / 680 / **799** | ≈ **0.000279** |

*Примечание:* латентность на стороне провайдера зависит от очереди и региона; один замер — ориентир, а не бенчмарк железа. Числа в таблице относятся к прогону с **короткой** формулировкой в переменной `PROMPT`; при **`-f prompt.txt`** длина промпта и счётчики токенов будут больше, выводы о качестве/соотношении моделей остаются сопоставимыми.

## Качество ответов

**Эталон:** \(P(\text{золотая}) = \frac{1}{3}\cdot 1 + \frac{1}{3}\cdot\frac{1}{2} + 0 = \frac{1}{2}\),  
\(P(A\mid\text{золотая}) = \frac{1\cdot 1/3}{1/2} = \mathbf{\frac{2}{3} \approx 66{,}7\%}\).

| Модель | Вердикт по задаче | Кратко |
|--------|-------------------|--------|
| **1B** | **Неверно** (получилось \(1/2\)) | Путаница в условной вероятности, нелогичные промежуточные рассуждения. |
| **7B** | **Верно** (\(2/3\)) | Явно Байес + полная вероятность, развёрнуто. |
| **72B** | **Верно** (\(2/3\)) | Структурированное решение, те же формулы, ответ совпадает с 7B. |

---

## Сравнение: скорость и ресурсоёмкость

- **Скорость:** самый быстрый ответ у **1B** (~2 с). **72B** оказалась быстрее **7B** в этом конкретном запуске, но обе «тяжёлые» модели сильно медленнее 1B в типичной картине; разброс по провайдеру может переворачивать порядок 7B vs 72B.
- **Токены (длина ответа):** **7B** сгенерировала больше всего completion-токенов (~739) — длиннее текст при той же задаче; **72B** чуть компактнее по токенам при похожей полноте вывода.
- **Стоимость:** выше у **72B** (~3.5× относительно 7B в этом замере). **1B** дешёва по API, но цена «ошибки» на задаче выше — ответ неверный.

---

## Короткий вывод

**Слабая (1B)** отвечает быстрее и дешевле по токенам/API, но на задаче с байесовской логикой **дала неверный численный ответ**. **Средняя (7B)** и **сильная (72B)** **сошлись на правильном \(2/3\)**: качество для этой задачи у 7B и 72B сопоставимо, а различия — в деталях изложения, длине генерации и **цене** (72B дороже). Для продакшена имеет смысл выбирать минимальную модель, которая **стабильно** держит нужный класс задач; здесь порог оказался между 1B и 7B.
