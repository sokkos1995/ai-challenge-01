🔥 День 20. Orchestration MCP

Зарегистрируйте несколько MCP-серверов.

Сделайте так, чтобы:

👉 агент выбирал нужный инструмент
👉 корректно маршрутизировал запросы
👉 выполнял длинный флоу взаимодействия

Проверьте:

👉 сценарий, в котором используются инструменты с разных серверов
👉 корректность выбора и порядка вызовов

Результат:

Длинный флоу взаимодействия с несколькими MCP-серверами и инструментами

## Реализация

Сделан отдельный оркестратор `homeworks/src/day_20_orchestration.py`, который:

1. Подключается к двум MCP-серверам:
   - `user-day-16-local-server` (`hello`, `add`);
   - `user-todoist-local` (`list_tasks`, `create_task`, `complete_task`).
2. На старте делает `initialize()` и `list_tools()` для каждого сервера.
3. Собирает единый registry `tool -> server` и валидирует, что нет конфликтов имен инструментов.
4. Выполняет длинный flow с роутингом между серверами:
   - `list_tasks` (Todoist) -> получить исходные данные;
   - `add` (day-16 server) -> вычислить производный метрик;
   - `create_task` (Todoist) -> сохранить результат обратно в Todoist.
5. Возвращает trace по шагам (какой инструмент вызван, на каком сервере и почему).

## Проверка

Тесты (`tests/test_day20_orchestration.py`) проверяют:

- сценарий с инструментами из разных серверов;
- корректность выбора сервера для каждого инструмента;
- корректность порядка вызовов (`list_tasks -> add -> create_task`);
- защиту от конфликтов имен инструментов в registry.

## Запуск

```bash
python3 homeworks/src/day_20_orchestration.py \
  --command "какие у меня задачи на сегодня и создай задачу-резюме"
```
