## Реализация

Сделан скрипт `homeworks/src/day_28_local_rag.py`.

Что реализовано:

- retrieval выполняется локально из индекса day 21 (`homeworks/artifacts/day_21/index_structured.json`);
- генерация ответа выполняется локальной моделью через Ollama (`/api/generate`);
- добавлено сравнение локальной и облачной модели (облако опционально через `--with-cloud`);
- рассчитываются метрики:
  - качество (`quality_keyword_coverage`);
  - скорость (`avg_latency_ms`);
  - стабильность (`avg_success_rate` + похожесть ответов при повторах).

Отчет сохраняется в:

- `homeworks/artifacts/day_28/local_rag_report.json`

## Как запустить

1) Убедиться, что есть индекс из day 21:

```bash
python3 homeworks/src/day_21_indexing.py
```

2) Запустить локальный Ollama:

```bash
ollama serve
```

3) Проверить локальный RAG (полностью локально):

```bash
python3 homeworks/src/day_28_local_rag.py \
  --index-path homeworks/artifacts/day_21/index_structured.json \
  --local-model qwen2.5:3b \
  --top-k 4 \
  --repeats 3
```

4) Запустить сравнение с облачной моделью (если есть ключи API):

```bash
python3 homeworks/src/day_28_local_rag.py \
  --index-path homeworks/artifacts/day_21/index_structured.json \
  --local-model qwen2.5:3b \
  --top-k 4 \
  --repeats 3 \
  --with-cloud
```

## Что именно сравнивается

Для набора контрольных вопросов:

- строится RAG prompt из локального retrieval;
- локальная модель запускается несколько раз (`--repeats`) для оценки стабильности;
- при `--with-cloud` тем же prompt тестируется облачный провайдер;
- в отчете фиксируются:
  - ответы обеих моделей;
  - ошибки вызовов;
  - время ответа;
  - итоговые усредненные метрики и дельта local vs cloud.
