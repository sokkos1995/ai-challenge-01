## Реализация (CLI-приложение на локальном Ollama)

Сделал небольшое приложение `homeworks/src/day_27_local_llm_app.py`, которое обращается к локальной модели через HTTP API Ollama.

В качестве локальной модели использованы результаты day 26:

- сервер: `http://localhost:11434`
- endpoint: `/api/generate`
- модель: `qwen2.5:3b`

Приложение работает в двух режимах:

1. **One-shot**: один запрос через аргумент `--prompt`
2. **Интерактивный чат**: последовательный диалог в консоли

Оба режима работают локально, без облачных LLM.

## Код приложения

Файл: `homeworks/src/day_27_local_llm_app.py`

Что делает скрипт:

- отправляет POST-запрос в `http://localhost:11434/api/generate`;
- передает payload:
  - `model: "qwen2.5:3b"`
  - `prompt: "<текст запроса>"`
  - `stream: false`
- парсит JSON-ответ и выводит поле `response` в консоль;
- обрабатывает ошибки недоступного сервера и невалидного ответа.

## Подготовка

1. Запустить локальный сервер Ollama:

```bash
ollama serve
```

2. Убедиться, что модель установлена:

```bash
ollama list
```

Ожидаемо в списке есть `qwen2.5:3b`.

## Запуск приложения

### Вариант 1: один запрос (one-shot)

```bash
python3 homeworks/src/day_27_local_llm_app.py \
  --prompt "Объясни в 3 пунктах, зачем использовать локальную LLM в проекте"
```

Пример ответа:

```text
1) Приватность: данные не покидают ваш контур.
2) Контроль: можно настраивать модель и инфраструктуру под задачу.
3) Предсказуемая стоимость: нет поминутной тарификации облачного API.
```

### Вариант 2: интерактивный чат

```bash
python3 homeworks/src/day_27_local_llm_app.py
```

Пример сессии:

```text
Local LLM chat started. Model: qwen2.5:3b
Type your question and press Enter. Type 'exit' to stop.

You: Что такое токены в LLM?
LLM: Токены — это базовые единицы текста, которые модель обрабатывает при анализе и генерации ответа.

You: Дай пример ограничения локальной модели.
LLM: Частое ограничение — более низкая скорость генерации на CPU по сравнению с мощными облачными GPU.

You: exit
Chat stopped.
```
